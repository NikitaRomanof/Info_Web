Функция №1
Функция, возвращает таблицу transferred_points в более человекочитаемом виде: Ник пира 1, ник пира 2, количество переданных пир поинтов. Количество переданных пойнтов может быть отрицательным, если пир 2 получил от пира 1 больше поинтов.

Функция №2
Функцию, возвращающая таблицу вида: ник пользователя, название проверенного задания, кол-во полученного XP. Таблица содержит только задания, успешно прошедшие проверку. Одна задача может быть успешно выполнена несколько раз. В таком случае таблица включает все успешные проверки.

Функция №3
Функция, определяющая пиров, которые не выходили из кампуса в течение всего дня Параметры функции: день, например 12.05.2022. Функция возвращает только список пиров.

Процедура №4
Процедура находит процент успешных и неуспешных проверок за всё время. Формат вывода: процент успешных проверок, процент неуспешных проверок.

Процедура №5
Процедура считает изменение в количестве пир-поинтов каждого пира по таблице transferred_points. Формат вывода: ник пира, изменение в количество пир-поинтов.

Процедура №6
Процедура считает изменения в количестве пир-поинтов каждого пира по таблице, возвращаемой первой функцией №1. Формат вывода: ник пира, изменение в количество пир-поинтов.

Процедура №7
Процедура находит самое часто проверяемое задание за каждый день. При одинаковом количестве проверок каких-то заданий в определенный день, будут выведены все задания. Формат вывода: день, название задания.

Процедура №8
Процедура определяет длительность последней P2P проверки. Под длительностью подразумевается разница между временем, указанным в записи со статусом "начало", и временем, указанным в записи со статусом "успех" или "неуспех". Формат вывода: длительность проверки.

Процедура №9
